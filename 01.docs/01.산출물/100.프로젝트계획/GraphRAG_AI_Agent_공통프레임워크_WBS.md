## 6. 진척 관리 기준

| 상태 | 기준 |
|---|---|
| 미착수 | 작업 시작 전 |
| 진행중 | 담당자가 작업을 시작했으나 산출물이 완료되지 않음 |
| 검토중 | 산출물 초안 작성 후 검토 진행 |
| 완료 | 산출물 작성 및 검토 완료 |
| 보류 | 외부 의사결정 또는 선행 작업 지연으로 진행 불가 |
| 변경 | 범위, 일정, 담당, 산출물이 변경됨 |

## 7. 변경 관리 기준

다음 항목이 변경될 경우 WBS를 갱신한다.

- 프로젝트 기간 또는 단계별 일정 변경
- 산출물 추가, 삭제, 통합, 예외 처리
- 파일럿 대상 프로젝트 변경
- Graph Store 또는 Vector Store 기술 스택 변경
- 주요 담당 역할 변경
- 완료 기준 변경

## 8. 관리 유의사항

- 본 WBS는 프로젝트 착수 기준 초안이며, 요구정의 완료 시점에 1차 기준선으로 확정한다.
- 구현 단계 이후 작업은 실제 설계 결과에 따라 세분화될 수 있다.
- GitHub 업로드는 PM 요청 시점에만 수행한다.
- 민감정보, API Key, DB Password, Token 등은 WBS 및 산출물에 기재하지 않는다.
- 산출물 변경 시 관련 요구사항추적표와 프로젝트계획서도 함께 영향도를 검토한다.
