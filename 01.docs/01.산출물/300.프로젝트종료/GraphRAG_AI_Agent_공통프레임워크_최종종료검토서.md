## 5. 잔여 리스크 및 후속 과제

| 구분 | 잔여 리스크/후속 과제 | 처리 방향 |
|---|---|---|
| 실제 저장소 통합 | PostgreSQL, PGVector, FAISS 운영 인스턴스 연계는 Adapter 골격 수준이다. | 후속 고도화에서 실제 저장소 통합 테스트 수행 |
| 관리자 E2E | 브라우저 기반 E2E 자동화는 설계/시나리오 수준으로 남아 있다. | Playwright 또는 Selenium 기반 회귀 테스트 구축 |
| 보안/권한 | 인증/인가 API와 역할별 접근 제어는 MVP 이후 보완 대상이다. | 관리자 권한 모델, 감사 로그, 토큰 정책 구체화 |
| 성능 기준 | 대용량 Source, 대규모 Chunk/Entity 기준 성능 벤치마크는 미수행이다. | 파일럿 데이터 확대 후 처리량/응답시간 기준 수립 |
| 운영 자동화 | 배포 파이프라인, 릴리즈 태그 생성, 운영 모니터링은 권고안 수준이다. | GitHub Actions, 릴리즈 태그, 운영 대시보드 구성 |
